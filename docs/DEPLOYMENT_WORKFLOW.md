# Website Deployment Workflow

## Phase 1: Security Verification (Tonight)
**Goal:** Prepare the environment for security testing.

### Steps
1. **Toggle for Testing**
   - [ ] Enable the target environment (e.g., Preview URL or Staging Build).
   - [ ] Ensure the site is reachable for the tester.

2. **Engage Security Tester**
   - [ ] Contact the security tester ("Hacker").
   - [ ] Provide the testing URL.
   - [ ] **Request:** "Please test security of the webpage."

## Phase 2: Go Live (Midnight / Post-MoU)
**Goal:** Public launch after MoU signing.

### Steps
1. **Midnight Toggle**
   - [ ] Turn on the main website (aureussigmacapital.com).
   - [ ] Remove any temporary access restrictions (e.g., password protection).

2. **Final Verification**
   - [ ] Check site loads correctly on public networks.
   - [ ] Verify SSL/HTTPS is active.
